# 핸드오프: 웨이브컷 WaveCut — 해수욕장 안전 서비스

## Overview
**웨이브컷(WaveCut)** 은 부산 해수욕장의 수심·조위·파고·이안류·날씨 공공데이터를 결합해, 시민이 해변 안전 정보를 직관적으로 이해하도록 돕는 AI 기반 해변 안전 서비스입니다. 핵심 기능은 **단면 수심 뷰** — 해변 평면도 위에서 단면선(모래사장 → 해안선 → 바다)을 드래그하면 해당 위치의 수심 변화를 **수직 단면도**로 보여주고, 수심을 숫자가 아닌 *물 없음 · 발목 · 무릎 · 허리 · 가슴 · 머리 이상* 의 **체감 단계**로 표현합니다.

대상: 부산시 공공데이터 AI 활용 경진대회 제출용 MVP. 웹(대시보드)과 앱(현장용 모바일) 두 표면을 포함합니다.

## About the Design Files
이 번들의 `design_files/` 안 파일들은 **HTML로 만든 디자인 레퍼런스(시제품)** 입니다 — 의도한 외형과 동작을 보여주는 프로토타입이며, 그대로 프로덕션에 복사해 쓰는 코드가 아닙니다. 작업 목표는 이 디자인을 **타깃 코드베이스의 환경(React/Next.js, Vue, React Native 등)에서 그 환경의 기존 패턴·라이브러리로 재현**하는 것입니다. 아직 코드베이스가 없다면 이 프로젝트에 가장 적합한 프레임워크(아래 권장 스택 참고)를 선택해 구현하면 됩니다.

현재 프로토타입은 브라우저에서 직접 Babel로 JSX를 변환하고, React/폰트를 CDN에서 불러오며, 데이터는 `data.js`에 하드코딩되어 있습니다. 이 세 가지(빌드 도구, 패키지 번들링, 실데이터 연동)가 프로덕션 전환 시 교체 대상입니다.

## Fidelity
**High-fidelity (hifi).** 최종 컬러·타이포그래피·간격·인터랙션이 모두 확정된 픽셀 단위 목업입니다. UI는 아래 디자인 토큰과 컴포넌트 명세를 그대로 따라 재현하세요. 단, 단면도(SVG)의 좌표·스케일 상수는 프로토타입 그대로 이식하면 동일하게 재현됩니다.

## 권장 구현 스택 (코드베이스가 없을 경우)
- **웹:** Next.js (App Router) + React + TypeScript. 공공 API 키 보호와 CORS 회피를 위해 **서버 라우트(route handler)에서 외부 API를 프록시**하세요. 브라우저에서 공공 API를 직접 호출하면 안 됩니다.
- **앱:** React Native(Expo) 또는 웹을 PWA로 패키징. 단면 뷰의 드래그 인터랙션은 RN에서 `PanResponder`/`react-native-gesture-handler` + `react-native-svg`로 재현.
- **단면도 렌더링:** SVG. 프로토타입의 `CrossSection.jsx` 기하 로직(좌표 변환·체감수심 컬럼·해저 polyline)을 거의 그대로 옮길 수 있습니다.
- **AI 코멘트:** 현재는 규칙 기반 문장 생성(`data.js`의 `aiComment`). 실서비스에서는 (1) 동일한 규칙 기반을 유지하거나 (2) 단면 분석 결과를 LLM에 넣어 자연어 코멘트를 생성하도록 서버에서 호출.

## 데이터 모델 & 핵심 로직 (`design_files/data.js`)
프로토타입의 전체 도메인 로직이 `data.js`(`window.BSM`)에 들어 있습니다. **이 파일은 비즈니스 로직 명세로 그대로 읽으세요.**

### 해변 데이터 (`BEACHES[]`)
부산 5개 해수욕장: 해운대, 광안리, 송정, 송도, 다대포. 각 객체 필드:
- 식별: `id`, `name`, `region`, `status`("safe"|"caution"|"danger"), `score`(0–100)
- **쉬운 정보(메인 카드/홈 노출):** `sky`(날씨 텍스트), `air`(기온℃), `uv`(자외선 "보통"|"높음"), `crowd`(예상 혼잡도 "여유"|"보통"|"많음"), `water`(수온℃)
- **전문 정보(상세 화면에서만 노출):** `wave`(파고 m), `tide`(조위 텍스트), `tideTrend`(상승/하강), `rip`(이안류 "안전"|"주의"|"위험"), `family`(가족 이용 추천 bool)
- 주차: `parking`(주차장명), `parkDist`(도보 거리)
- `summary`(한 줄 요약), `transects[]`(단면 프로파일 — 아래)

### 체감 수심 단계 (`LEVELS`)
| key | label | 깊이 임계(m, 조위0 기준) | status |
|---|---|---|---|
| none | 물 없음 | depth ≤ 0.02 | safe |
| ankle | 발목 | ≥ 0 | safe |
| knee | 무릎 | ≥ 0.3 | safe |
| waist | 허리 | ≥ 0.6 | caution |
| chest | 가슴 | ≥ 1.0 | caution |
| head | 머리 이상 | ≥ 1.5 | danger |

`levelOf(depth)` → 깊이(m)를 받아 해당 단계 반환.

### 시간대 / 조위 시뮬레이션 (`TIMES`)
3단계 고정(MVP): `현재`(14:00, 조위 +0.0m), `1시간 후`(15:00, +0.35m), `2시간 후`(16:00, +0.70m). **3시간 후 이상은 MVP에 넣지 않습니다.** 조위값은 해당 시각의 수심에 단순 가산됩니다.

### 단면 프로파일 (`transects[]` + `bedDepth`)
각 해변은 여러 단면선 지점(좌/중/우)에 대한 파라미터 배열을 가집니다:
- `shelf`: 완만한 모래턱 길이(m)
- `shelfDepth`: 모래턱 끝 깊이(m)
- `slope`: 그 이후 급경사 기울기(m 깊이 / m 거리)
- `rip`: 해당 구간 이안류 상태

해안선으로부터 거리 `d`(음수=마른 모래사장)에 대한 해저 깊이:
```
d < 0          → (d / 14) * 0.5           // 마른 모래사장
d ≤ shelf      → (d / shelf) * shelfDepth  // 완만한 모래턱
d > shelf      → shelfDepth + (d - shelf) * slope  // 급경사
```
`transectAt(beach, p)`: 단면선 위치 `p`(0~1)를 받아 인접 두 transect를 선형 보간 → 드래그 시 연속적으로 단면이 변함.
`depthAt(beach, p, tideKey, d)`: `bedDepth(d) + 조위보정` = 체감 수심.
`analyze(beach, p, tideKey)`: 해안선에서 무릎끝(depth>0.6) 거리 = `kneeEnd`, 위험시작(depth≥1.5) 거리 = `dangerStart` 반환.

### AI 안전 코멘트 (`aiComment`)
2줄 요약형. 위치명(좌측/중앙/우측) + `kneeEnd`까지 무릎 수심이라 가족 적합 + `dangerStart` 이후 급경사 주의. 예:
> "현재 선택한 중앙 단면은 해안선에서 약 30m까지 무릎 수심으로 가족 이용에 적합합니다. 45m 이후부터 수심이 빠르게 깊어지므로 어린이와 초보자는 주의가 필요합니다."

### 상황별 권장 행동 (`situationTips`)
4개 카드 고정: 가족 동반 / 수영 초보자 / 오후 방문 예정 / 혼잡 시간 방문. 각 카드는 status(safe/caution) + 제목 + 짧은 행동 가이드. 분석 결과(`kneeEnd`, `dangerStart`, `crowd`)에 따라 톤과 문구가 달라집니다.

## 정보 위계 원칙 (중요)
**메인/홈 카드 = 쉬운 정보만, 상세 화면 = 전문 안전 정보.** 이 분리는 디자인의 핵심 의사결정이니 재현 시 반드시 지키세요.
- 카드/홈: 해수욕장명, 날씨+기온, 자외선, 예상 혼잡도, 안전 상태 배지, 상세 보기 진입
- 카드에 **노출 금지:** 파고, 조위, 이안류, 중조/소조/대조, 가족 추천 여부
- 상세 화면: 위 쉬운 정보 + 주차(버튼형) + 파고/조위/이안류/가족추천(전문 분석) + "단면 수심 보기" 진입
- 단면 수심 뷰 화면 **하단:** AI 안전 코멘트 + 상황별 권장 행동 (기본 상세 화면이 아니라 단면 뷰 안에)

## 단면 수심 뷰 컴포넌트 명세 (`CrossSection.jsx`) — 가장 중요
3개 영역으로 구성:

### (1) 시간대 탭 (조위 시뮬레이션)
- 라벨 "조위 시뮬레이션" + 세그먼트 컨트롤(현재/1시간 후/2시간 후).
- **버튼 통일 규칙:** 3버튼 균등 너비(flex:1), 동일 높이 42px, 동일 radius 8px, 동일 padding. 선택 버튼만 블루(`--blue-600`) 배경 + 흰 텍스트, 미선택은 흰 배경 + 옅은 보더. 버튼 간격 6px 균일. 각 버튼은 라벨 + 작은 시각(em).

### (2) 평면도 (단면선 드래그)
- 가로 3밴드: 모래사장(상) / 해안선(중) / 바다(하, 얕은바다←→깊은바다 라벨).
- 좌/중앙/우 틱 표시. 세로 단면선(`plan-line`)을 포인터로 드래그 → 위치 `p`(0.02~0.98) 갱신. 핸들(knob)에 좌우 화살표 아이콘 + "단면선" 캡션.
- 인터랙션: `onPointerDown`으로 드래그 시작, window `pointermove`/`pointerup`로 추적. (RN에서는 PanResponder로 재현)

### (3) 수직 단면도 (SVG)
- viewBox `0 0 820 {300|340}`. 좌표 상수: `X0=56, X1=802, D_MIN=-12, D_MAX=80, SURFACE_Y=64, SCALE=(VB_H-34-SURFACE_Y)/2.4`.
- `xOf(d)`: 거리(m)→x, `yOf(depth)`: 깊이(m)→y.
- 렌더 순서: 수면 사각형(`--sky-50`) → 체감수심 컬럼(2m 간격, 각 컬럼 색 = `levelOf(depth).color`) → 해저 채움(`#E4D2B0` 모래색) + 해저선(`#B79B68`, 2px) → 수면선(`#2f86f0`, 2px) → Y축 가이드선(발목/무릎/허리/가슴·위험) → 위험시작 표시(빨강 점선 + "Nm 급경사" 배지) → 추천 입수 구간(초록 선 "추천 입수 구간 0–Nm") → 거리 눈금(0/20/40/60/80m).
- 하단: 체감수심 범례(6단계 컬러 칩).

### (4) AI 안전 코멘트 카드 (단면 뷰 하단)
- "AI" 배지 + 제목 + "실시간 해석" 라이브 닷 + 2줄 본문 + 출처 푸터("수심·조위·파고·이안류 데이터를 종합한 추정 결과입니다.").

## Interactions & Behavior
- **단면선 드래그:** 평면도에서 포인터 드래그 → `p` 갱신 → transect 보간 → 단면도 실시간 리렌더 + AI 코멘트 문구 갱신. 클램프 0.02~0.98.
- **시간대 전환:** 탭 클릭 → `tideKey` 변경 → 모든 깊이에 조위 가산 → 단면도/체감단계 갱신.
- **웹↔앱 토글:** 상단 chrome 토글(프로토타입 전용 데모 장치 — 실제 서비스에선 불필요).
- **카드/리스트 클릭:** 상세 화면 진입. 상세 "단면 수심 보기" → 단면 뷰 진입.
- **애니메이션:** 절제. 180ms ease-out 페이드/색 전환. 바운스·무한 루프 없음.
- **상태/로딩/에러:** 프로토타입엔 없음. 실서비스에서 API 로딩 스켈레톤 + 데이터 없음/실패 상태 추가 필요.

## 실데이터 연동 가이드 (프로덕션 전환)
`data.js`의 하드코딩 값을 아래 공공 API로 대체:
- **국립해양조사원:** 조위(실시간/예보), 수온, **수심측량/해저지형**(단면 프로파일의 실제 근거 — 가장 확보가 까다로움, 사전 확인 권장).
- **기상청:** 날씨, 기온, 자외선 지수.
- **해양경찰청 / 관련 기관:** 이안류 정보.
- data.go.kr에서 API 키 발급 → **서버(Next.js route handler 등)에서 호출**(키 보호 + CORS 회피) → transect 파라미터(shelf/shelfDepth/slope)는 수심측량 데이터로부터 산출하거나, 없으면 현 프로토타입 값을 시드로 사용.
